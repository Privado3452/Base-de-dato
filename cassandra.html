Cerrar Sesión

Es una base de datos NoSQL distribuida y de código abierto diseñada para manejar grandes volúmenes de datos estructurados y no estructurados de manera eficiente y escalable.




Caracteristicas

  • Alta escalabilidad: Cassandra está diseñada para distribuirse en múltiples nodos, lo que le permite escalar horizontalmente para manejar grandes volúmenes de datos y tráfico.

  • Alta disponibilidad: Cassandra replica los datos en varios nodos, lo que garantiza que los datos estén disponibles incluso si un nodo falla.

  • Bajo costo: Cassandra es una base de datos de código abierto y gratuita, lo que la convierte en una opción rentable para organizaciones con presupuestos limitados.

  • Consistencia eventual: Cassandra ofrece una consistencia eventual, lo que significa que los cambios en los datos se propagan a todos los nodos del clúster en un período de tiempo, no de forma instantánea.




Funcionamiento

Cassandra almacena datos en clusters distribuidos, compuestos por varios nodos. Cada nodo almacena una partición de los datos. Los datos se dividen en particiones mediante una clave de partición. Las réplicas de las particiones se almacenan en varios nodos para garantizar la disponibilidad.




Desventaja

  • Complejidad: Cassandra puede ser más compleja de administrar que las bases de datos relacionales tradicionales.

  • Aprendizaje: La curva de aprendizaje de Cassandra puede ser más pronunciada que la de las bases de datos relacionales tradicionales.

  • Consistencia eventual: La consistencia eventual de Cassandra puede no ser adecuada para aplicaciones que requieren una consistencia fuerte de los datos.
